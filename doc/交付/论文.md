### 摘要

临床实习是医学高等教育中连接理论知识与临床实践的核心环节。然而，传统医学实习管理模式长期面临着学校与实习医院之间信息传递滞后、实习过程难以量化追踪、以及多角色协同效率低下的“信息孤岛”问题。为解决上述痛点，本文设计并实现了一套基于前后端分离架构的医校联动实习管理系统。

本系统在后端采用 Java 17 与 Spring Boot 框架构建 RESTful API 服务，利用 Spring Data JPA 规范化数据持久层操作，并自主实现了一套基于拦截器与线程上下文（ThreadLocal）的轻量级 RBAC（基于角色的访问控制）权限校验机制。前端则基于 Vue 3 渐进式框架与 Element Plus 组件库，构建了响应式且交互友好的单页面应用（SPA）。在业务维度上，系统深度切合医校协同场景，抽象出系统管理员、学校管理员、医院管理员、带教老师及学生五类核心角色。实现了涵盖“医院发岗、学生申请、多级审批、过程追踪（周记与任务）、系统通知、双向评价”的全生命周期闭环管理。

经过严格的功能与集成测试，系统运行稳定可靠，有效打破了校院两端的壁垒，实现了实习数据的透明化与可追溯化，对提升医学实习生管理质量具有较高的实践应用价值。

**关键词：** 医校联动；实习管理系统；Spring Boot；Vue 3；前后端分离；RBAC

### Abstract

Clinical internship is the core link connecting theoretical knowledge with clinical practice in higher medical education. However, the traditional medical internship management mode has long faced the "information silo" problem, characterized by delayed information transfer between medical schools and hospitals, difficulties in quantitatively tracking the internship process, and low efficiency in multi-role collaboration. To address these pain points, this paper designs and implements a medical-school linked internship management system based on a front-end and back-end separation architecture.

The backend of this system adopts Java 17 and the Spring Boot framework to construct RESTful API services. It utilizes Spring Data JPA to standardize data persistence operations and implements a lightweight Role-Based Access Control (RBAC) mechanism based on interceptors and ThreadLocal context. The frontend leverages the Vue 3 progressive framework and Element Plus UI library to build a responsive and interactive Single Page Application (SPA). In terms of business logic, the system abstracts five core roles: System Administrator, School Administrator, Hospital Administrator, Teaching Teacher, and Student. It realizes a closed-loop management of the entire internship lifecycle, covering "hospital post publication, student application, multi-level approval, process tracking (weekly reports and tasks), system notification, and two-way evaluation".

Through rigorous functional and integration testing, the system operates stably and reliably, effectively breaking the barriers between schools and hospitals, and realizing the transparency and traceability of internship data. It holds significant practical application value for improving the management quality of medical interns.

**Keywords:** Medical-School Linkage; Internship Management System; Spring Boot; Vue 3; Front-end and Back-end Separation; RBAC

### 第一章 绪论

#### 1.1 研究背景与意义

在当今高等医学教育体系中，临床实习被视为医学生向合格临床医师转型的关键“孵化期”。教育部及卫健委等多项指导文件均强调了临床实践质量控制的重要性。然而，在实际运行过程中，医学院校与各大附属或教学医院分属不同的行政管理体系，导致实习管理工作长期处于一种松散、割裂的状态。

传统的实习管理多依赖于纸质表格的流转或简单的电子表格汇总，这种模式暴露出诸多弊端：首先是“信息孤岛”现象严重，学校难以实时掌握学生在医院的具体科室流转与考勤状况；其次是实习过程缺乏有效的质量监控，带教老师与学生之间的互动（如任务布置、周记批阅）多在线下或通过即时通讯软件碎片化进行，难以形成系统性的教学档案；最后是评价机制单一，往往只有实习期末的一纸鉴定，缺乏动态的双向反馈机制。

在此背景下，借助现代信息技术手段，打破医学院校与医院之间的数据壁垒，研发一套高度契合医学实习特性的联动管理系统显得尤为迫切。本系统的研发意义在于：一方面，通过信息化的审批流与数据追踪，极大减轻教务人员与医院科教处的工作负担，提升管理协同效率；另一方面，借助全程留痕的任务记录与评价模型，为医学实习质量评估提供客观的数据支撑，最终助力医学人才培养质量的整体提升。

#### 1.2 国内外研究现状

近年来，随着教育信息化的推进，国内外诸多高校已逐步引入实习管理信息系统。在国外，医学教育起步较早的地区普遍采用了功能较为庞大的教务综合平台，这类平台通常内嵌了详尽的临床技能轮转登记（Logbook）模块，具备较高的定制化程度，但其高昂的维护成本以及与国内医学教育体制的差异，使得直接引进的可行性极低。

国内的实习管理系统起步稍晚，目前市场上已出现部分通用型的大学生实习就业跟踪平台。然而，大多数现有系统主要面向泛工科或文科专业，业务逻辑侧重于“企业-学校-学生”的三方协议签订与最终报告提交，缺乏对医学实习独有的“多科室轮转、高频次带教互动（周记/病例分析）、多维度评价”特性的支持。此外，部分医院内部虽然建有科教管理系统，但往往无法对接到学校端，使得学校依然面临数据盲区。因此，开发一款专属定制、聚焦于“医校联动”与“过程化管理”的轻量级实习管理系统，是当前该领域研究的重要趋势。
