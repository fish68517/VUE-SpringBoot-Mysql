# （1）主要研究内容：

## 系统功能分析与需求设计：

分析用户需求，明确前端与后台的功能设计，包括用户注册与登录、活动报名系统、众筹资金相关机制、活动统计反馈、以及系统管理员对用户、活动和众筹资金的管理功能等。

## 活动与众筹协作模块的设计与实现：

设计活动发布与管理功能，支持组织者自定义活动的名称、类型、时间和地点等相关信息；开发众筹协作核心逻辑，允许活动组织者按需设定众筹目标金额，实时计算并展示众筹完成率；实现众筹支付功能，对接统一支付接口，保障支付安全；设计活动进度展示模块，通过进度条、数据卡片等形式，向组织者和参与者实时同步报名人数、众筹金额等核心数据。

## 反馈与数据统计模块的设计与实现：

设计活动反馈模块，支持参与者在活动结束后对活动进行文字反馈，反馈内容自动关联对应活动；开发数据统计与可视化功能，为组织者提供活动参与情况、众筹完成率等核心数据的图标展示（如柱状图、折线图）。


**活动报名与管理模块**

活动组织者在完成登录后，进入“活动管理-创建活动”页面，填写活动基础信息：活动名称、类型、时间、地点、报名截至时间、参与人数限制等；若活动需要众筹支持，可设置众筹金额目标金额，支持自定义档位与金额。活动信息提交后，需要等待管理员审核，审核通过后自动发布至系统首页活动列表。

普通用户在浏览活动详情时，可以点击每一项活动的“立刻报名”按钮去填写报名表单，从而完成活动的报名操作。前端通过Vue3表单校验检测（如手机号格式、学号唯一性）拦截无效数据，后端接口二次校验（查询数据库判断是否重复报名），在报名时间段内，参与者能自主进行一个退出活动的操作，当超出这个时间段后，则需要与活动组织者联系，才能完成退出活动的操作。为了方便参与者和组织者了解活动的参与情况，前端的活动详情页通过可视化组件展示核心进度数据，如：报名进度（已报名人数/总限制人数，以进度条呈现）、众筹进度（已筹金额/目标金额，显示百分比与进度条，标注支持者人数）等，当这些核心数据发生变动时，前端能通过Axios异步请求更新页面，无需手动刷新。

**众筹协作模块**

普通用户浏览活动详情时，若活动开启众筹，可查看各个支持档位，选择对应档位后点击“支持众筹”后会跳转到支付页面，支付对接校园统一支付接口（如校园卡一卡通），避免直接处理支付敏感信息。完成支付后，后端记录众筹支持记录（用户ID、活动ID、支付金额、支付时间等），并实时更新活动众筹进度，并且支持者能在“我的众筹”页面查看支持记录等信息。

管理员端可以查看所有活动的众筹资金明细，包括总筹金额、支持人次、资金流向等，组织者需在活动结束7天后，提交众筹资金使用情况（如上传发票凭证等），经管理员审核通过后，方能完成资金结算，并且具体的资金使用情况会显示在活动的“资金使用公示”入口，所有用户可查看对应活动的资金使用明细，保障资金去向和使用情况的透明度。

**反馈与数据统计模块**

活动结束后，系统将自动触发反馈提醒机制，向该活动的所有参与者推送反馈通知；参与者可通过通知入口进入反馈页面，完成星级评价（1-5星）与文字反馈的提交。对于活动组织者，系统会整合其名下活动的全维度数据生成反馈汇总报表，涵盖报名情况（报名人数）、众筹情况（众筹完成率、支持人次）、反馈核心数据（平均评分、评分分布）等内容，并基于ECharts实现数据可视化展示（如评分柱状图、众筹进度折线图）。同时，报名、众筹、反馈等操作产生的实时数据，会通过前端异步请求实时同步至后端统计模块，经数据校验后更新至数据库，确保统计结果的准确性与时效性；组织者还可在反馈汇总页面直接回复参与者的文字反馈，实现与参与者的高效互动。


## 管理端核心功能的设计与实现：

开发活动审核模块，设计合理的审核流程，支持管理员查看活动详情信息，通过“通过”“驳回（附理由）”操作，审核结果实时通知组织者；设计用户管理功能，支持管理员查看系统所有用户信息，进行账号禁用/启用操作；开发管理员设置活动分类、报名表单模板等基础参数，灵活适配校园活动管理需求。



# 校园活动众筹与报名平台——详细功能点清单

## 一、普通用户功能模块

### 1. 修改个人信息

- 基础信息编辑：修改昵称、头像、性别、生日、联系方式（手机号/邮箱）
- 账号安全：修改登录密码、绑定/解绑第三方账号（微信/QQ）、设置账号隐私权限
- 身份认证：提交学生/教职工身份认证材料（如学号、工号），提升账号可信度

### 2. 浏览活动

- 活动筛选：按活动类型（讲座/比赛/公益）、时间（近期/往期）、热度（参与人数/众筹进度）、主办方等维度筛选
- 详情查看：查看活动封面、介绍、目标金额、众筹进度、报名人数、时间地点、参与要求、主办方信息
- 收藏/分享：收藏感兴趣的活动，生成分享链接/海报分享至社交平台

### 3. 活动报名/退出

- 报名操作：填写报名信息（姓名/联系方式/备注），提交报名申请；支持批量报名多活动
- 状态查询：查看报名审核状态（待审核/已通过/已拒绝）、活动签到码
- 退出操作：在活动开始前取消报名，支持自动退款（若已缴纳报名费）

### 4. 众筹支持

- 金额选择：支持自定义金额或预设档位（如10元/50元/100元）进行支持
- 支付方式：对接微信/支付宝/校园卡等支付渠道，生成订单并完成支付
- 进度查看：查看众筹项目实时进度、自己的支持记录、回报权益（如纪念品/感谢信）

### 5. 活动反馈

- 评价提交：活动结束后对活动体验、主办方服务、众筹透明度进行星级评分+文字评价
- 图片/视频上传：上传活动现场照片/视频，丰富反馈内容
- 反馈查看：查看其他用户对同一活动的评价，作为后续参与参考

---

## 二、活动组织者功能模块

### 1. 添加活动

- 信息录入：填写活动名称、类型、封面、详细介绍、时间地点、参与人数上限、报名要求
- 规则设置：设置报名截止时间、是否需要审核、报名费金额（可选）
- 草稿保存：支持保存未完成的活动信息，后续继续编辑

### 2. 众筹发布

- 众筹配置：设置目标金额、众筹周期、回报方案（如不同档位的纪念品/权益）
- 资金管理：绑定收款账户，设置资金提现规则（如众筹成功后分阶段提现）
- 进度更新：发布众筹进度动态，向支持者同步项目进展

### 3. 修改活动信息

- 信息更新：修改活动时间、地点、参与要求、众筹目标等信息
- 变更通知：修改后自动推送通知给已报名/支持的用户
- 撤回操作：在活动未开始前撤回已发布的活动，支持退款处理

### 4. 提供资金流向证明

- 材料上传：上传发票、收据、采购清单等资金使用凭证，标注使用场景
- 公开透明：设置证明材料的公开范围（仅对支持者可见/全平台可见）
- 进度同步：按阶段提交资金使用报告，如众筹中期/结束后

---

## 三、后台管理人员功能模块

### 1. 审核活动信息

- 审核流程：对新提交/修改的活动信息进行合规性审核（内容真实性、是否违反平台规则）
- 审核操作：通过/驳回申请，驳回时填写具体原因并反馈给主办方
- 批量处理：支持批量审核多个活动，提升效率

### 2. 审核资金流向证明

- 材料核验：检查资金使用凭证的真实性、合规性，核对金额与用途
- 审核反馈：通过/驳回证明材料，驳回时要求主办方补充或修改
- 公示管理：审核通过的证明材料自动在平台公示，保障透明度

### 3. 活动管理

- 全量查看：查看平台所有活动的状态（待审核/进行中/已结束/已下架）
- 违规处理：对违规活动（虚假宣传、非法集资等）进行下架、封禁主办方权限
- 数据统计：统计活动参与人数、众筹金额、用户反馈等数据，生成报表

### 4. 用户管理

- 账号管理：查看所有用户信息，对违规账号（恶意评价、虚假报名）进行封禁/警告
- 权限分配：为不同角色（普通用户/组织者/管理员）分配对应操作权限
- 数据导出：导出用户数据报表，用于平台运营分析
