# **系统需求分析**

本章主要对基于Android的企业仓库管理系统进行详细的需求分析。通过对系统目标的阐述，明确了开发此系统的必要性。通过对用户的角色和业务流程的分析，我们得出了系统的功能性需求和非功能性需求，这将作为后续系统设计与实现的重要依据。

## **1.1 项目背景与目标**

随着企业规模的不断扩大，传统的人工仓库管理模式暴露了诸多弊端，例如：数据记录不准确、库存信息更新不及时、出入库操作效率低下、货物盘点耗时耗力等。这些问题严重影响了企业的运营效率和成本控制。

为了解决上述痛点，本项目旨在开发一个基于Android移动端和Web后端相结合的现代化企业仓库管理系统。系统通过引入二维码技术，实现了对货物从入库、出库到库存查询的全流程数字化管理。操作人员使用Android App扫描二维码即可完成出入库操作；管理人员则通过Web后端随时监控库存状态、管理基础数据并进行数据统计分析。

本系统的核心目标是：

● **提升效率**：通过扫码出入库，简化操作流程，大幅提升作业效率。  
● **确保准确**：减少人工录入环节，避免人为错误，保证库存数据的实时性和准确性。  
● **降低成本**：实现无纸化办公，节省人力和物料成本，并通过精确的库存管理减少资金占用。  
● **优化决策**：提供多维度的数据查询和统计报表，为管理层的采购和销售决策提供数据支持。

## **1.2 用户角色分析**

本系统主要面向三类用户，他们各自拥有不同的职责和操作权限：

1\. **仓库管理员 (Warehouse Manager)**

○ **职责**：负责仓库的日常单据管理和库存监控。和权限分配  
○ **主要操作**：在Web后端进行入库清单管理（如通过Excel文件批量导入入库商品信息）、为商品或批次生成并打印二维码、查询实时库存、处理异常出入库记录、查看数据统计报表。  
2\. **仓库操作员 (Warehouse Operator)**

○ **职责**：负责仓库内货物的具体出入库执行操作。  
○ **主要操作**：使用手持Android设备（App）登录个人账号，通过扫描货物上的二维码执行入库、出库操作。操作简单、直接。

## **1.3 功能性需求分析**

功能性需求是系统必须实现的核心功能，是整个项目的基石。本系统分为 **Web后端管理平台** 和 **Android移动端App** 两部分。

### **1.3.1 基础通用功能**

这些功能是Web后端和App都具备或相关联的基础模块。

| 功能模块 | 功能描述 |
| :---- | :---- |
| **用户登录** | \- 提供登录界面，用户凭账号和密码登录系统。 \- 系统根据用户角色（系统管理员、仓库管理员、操作员）分配不同的操作界面和权限。 \- 提供“记住密码”功能。 |
| **退出登录** | \- 用户可以随时安全退出系统，清除登录状态。 |
| **修改密码** | \- 用户登录后，可以在个人中心修改自己的登录密码，增强账户安全性。 |

### **1.3.2 Web后端管理平台主要功能**

| 功能模块 | 功能描述 |
| :---- | :---- |
| **仪表盘 (Dashboard)** | \- 系统首页，以图表形式（如柱状图、饼图）直观展示核心数据，如：今日出入库数量、库存总量、库存预警信息等。 |
| **用户管理** | \- (仅系统管理员) 实现对系统所有用户的增、删、改、查操作。 \- 管理用户信息，包括账号、姓名、所属部门、角色等。 \- 可重置用户密码。 |
| **部门管理** | \- (仅系统管理员) 对企业的部门结构进行管理，支持部门的增、删、改、查。 |
| **入库管理** | \- 支持手动创建入库单。 \- 核心功能：支持通过Excel/CSV模板批量导入入库商品清单，系统自动解析并生成入库任务。 \- 核心功能：为生成的入库单或单个商品生成包含唯一标识信息的二维码，并支持批量打印。 |
| **出库管理** | \- 主要用于查看由App端操作的出库记录流水。 \- 支持手动处理或登记特殊的出库情况。 |
| **库存查询** | \- 提供强大的库存查询功能，可根据商品名称、编号、批次、入库时间等多种条件进行筛选和查询。 \- 实时显示商品库存数量、位置等信息。 \- 支持将查询结果导出为Excel文件。 |
| **数据统计** | \- 提供多维度的数据统计报表。 \- 例如：按时间范围（日、周、月）统计出入库流水；按商品统计库存周转率等。 \- 以图表和数据表格结合的方式展示，方便决策分析。 |

### **1.3.3 Android移动端App主要功能**

| 功能模块 | 功能描述 |
| :---- | :---- |
| **扫码入库** | \- 操作员登录后，选择“扫码入库”功能，调用摄像头。 \- 扫描待入库商品上的二维码，App解析二维码信息并显示商品详情。 \- 操作员确认信息无误后，点击“确认入库”，数据通过网络API提交至后端服务器，对应商品库存增加。 |
| **扫码出库** | \- 操作员选择“扫码出库”功能，调用摄像头。 \- 扫描待出库商品上的二维码，App显示商品详情。 \- 操作员确认信息并提交，后端服务器更新库存，商品库存减少，并生成出库记录。 |
| **数据查询** | \- 提供简单的商品信息查询功能。 \- 操作员可以通过扫描二维码或输入商品编号，快速查看该商品的当前库存量等基本信息。 |

## **1.5 界面原型设计描述**

根据功能需求，这里对几个核心界面的设计进行文字描述。

1\. **登录界面 (App & Web)**

○ **布局**：垂直居中布局。  
○ **元素**：顶部为系统Logo和标题“企业仓库管理系统”，中间为两个输入框，分别用于输入“账号”和“密码”，密码框带有可见/隐藏切换按钮。下方为一个“登录”按钮。  
2\. **Web后端主界面**

○ **布局**：经典的后台管理系统左右布局。  
○ **左侧**：为固定导航菜单栏，包含“仪表盘”、“入库管理”、“出库管理”、“库存查询”、“用户管理”、“部门管理”等模块入口。  
○ **右侧**：为内容显示区。点击左侧菜单，右侧会相应地展示数据表格、表单或统计图表。  
3\. **Android App主界面**

○ **布局**：简洁的九宫格或列表布局。  
○ **元素**：屏幕中央放置几个尺寸较大、图标清晰的按钮，如“扫码入库”、“扫码出库”、“数据查询”。底部为导航栏或顶部为标题栏，显示当前登录的用户名，并提供“修改密码”和“退出登录”的入口。  
4\. **App扫描界面**

○ **布局**：全屏的摄像头预览。  
○ **元素**：屏幕中央有一个透明的矩形取景框，引导用户对准二维码。提供一个打开闪光灯的按钮。扫描成功后，预览界面会弹出一个半透明的浮层，显示商品名称、规格、数量等信息，并附带“确认”和“取消”按钮。

## **1.7 数据库设计**

\-- \=============================================

\-- 1\. 部门表

\[cite\_start\]-- 用于管理组织架构 \[cite: 25\]。

\[cite\_start\]-- 对应 Web 端的 "部门管理" 功能 \[cite: 35\]。

\-- \=============================================

CREATE TABLE \`departments\` (

  \`id\` INT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`name\` VARCHAR(100) NOT NULL COMMENT '部门名称',

  \`created\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP,

  \`updated\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP,

  UNIQUE KEY \`uk\_name\` (\`name\`)

) COMMENT='部门信息表';

 

\-- \=============================================

\-- 2\. 角色表

\-- 定义用户角色及其权限。

\[cite\_start\]-- 系统预设了系统管理员、仓库管理员、操作员等角色 \[cite: 12\]。

\-- \=============================================

CREATE TABLE \`roles\` (

  \`id\` INT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`role\_name\` VARCHAR(50) NOT NULL COMMENT '角色名称 (例如: 系统管理员, 仓库管理员)',

  UNIQUE KEY \`uk\_role\_name\` (\`role\_name\`)

) COMMENT='用户角色表';

 

\-- \=============================================

\-- 3\. 用户表

\-- 存储用户账户信息。

\[cite\_start\]-- 支持用户登录、修改密码和用户管理功能 \[cite: 23, 25\]。

\-- \=============================================

CREATE TABLE \`users\` (

  \`id\` INT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`username\` VARCHAR(50) NOT NULL COMMENT '登录用户名',

  \`password\_hash\` VARCHAR(255) NOT NULL COMMENT '加密后的密码',

  \`full\_name\` VARCHAR(100) COMMENT '用户全名',

  \`role\_id\` INT UNSIGNED NOT NULL COMMENT '关联角色表的外键',

  \`department\_id\` INT UNSIGNED COMMENT '关联部门表的外键',

  \`is\_active\` BOOLEAN NOT NULL DEFAULT TRUE COMMENT '标记用户账户是否启用',

  \`created\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP,

  \`updated\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP,

  UNIQUE KEY \`uk\_username\` (\`username\`),

  KEY \`idx\_role\_id\` (\`role\_id\`),

  KEY \`idx\_department\_id\` (\`department\_id\`),

  CONSTRAINT \`fk\_users\_role\` FOREIGN KEY (\`role\_id\`) REFERENCES \`roles\` (\`id\`) ON DELETE RESTRICT,

  CONSTRAINT \`fk\_users\_department\` FOREIGN KEY (\`department\_id\`) REFERENCES \`departments\` (\`id\`) ON DELETE SET NULL

) COMMENT='系统用户账户表';

 

\-- \=============================================

\-- 4\. 产品表 (商品表)

\-- 存储仓库中所有货品的基础信息。

\-- 是库存查询、出入库管理的基础。

\-- \=============================================

CREATE TABLE \`products\` (

  \`id\` INT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`sku\` VARCHAR(100) NOT NULL COMMENT 'SKU (Stock Keeping Unit), 产品的唯一标识符',

  \`name\` VARCHAR(200) NOT NULL COMMENT '产品名称',

  \`description\` TEXT COMMENT '产品详细描述',

  \`created\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP,

  \`updated\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP,

  UNIQUE KEY \`uk\_sku\` (\`sku\`),

  KEY \`idx\_name\` (\`name\`)

) COMMENT='产品主数据表';

 

\-- \=============================================

\-- 5\. 入库单表

\[cite\_start\]-- 管理入库任务，通常由Excel批量导入创建 \[cite: 25\]。

\-- 这是一个入库批次的头信息。

\-- \=============================================

CREATE TABLE \`inbound\_orders\` (

  \`id\` INT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`order\_number\` VARCHAR(50) NOT NULL COMMENT '入库单的唯一编号',

  \[cite\_start\]\`created\_by\_user\_id\` INT UNSIGNED NOT NULL COMMENT '创建该订单的用户 (仓库管理员) \[cite: 15\]',

  \`status\` ENUM('待处理', '处理中', '已完成') NOT NULL DEFAULT '待处理' COMMENT '入库单状态',

  \`notes\` TEXT COMMENT '额外备注',

  \`created\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP,

  \`updated\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP,

  UNIQUE KEY \`uk\_order\_number\` (\`order\_number\`),

  KEY \`idx\_created\_by\_user\_id\` (\`created\_by\_user\_id\`),

  CONSTRAINT \`fk\_inbound\_orders\_user\` FOREIGN KEY (\`created\_by\_user\_id\`) REFERENCES \`users\` (\`id\`) ON DELETE RESTRICT

) COMMENT='入库单 (任务) 表';

 

\-- \=============================================

\-- 6\. 库存表

\-- 系统的核心表，代表了实际的库存。每条记录都是一个特定批次的产品。

\[cite\_start\]-- 二维码将唯一标识此表中的一条记录，用于所有扫码操作 \[cite: 5\]。

\-- \=============================================

CREATE TABLE \`inventory\` (

  \`id\` INT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`product\_id\` INT UNSIGNED NOT NULL COMMENT '关联产品表的外键',

  \`batch\_code\` VARCHAR(100) NOT NULL COMMENT '批次唯一编码, 用于生成二维码',

  \`inbound\_order\_id\` INT UNSIGNED COMMENT '此批次所属的入库单',

  \`quantity\` INT NOT NULL DEFAULT 0 COMMENT '该批次的当前库存数量',

  \`received\_at\` DATETIME COMMENT '批次被实际扫码入库的时间',

  \`created\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP,

  \`updated\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP,

  UNIQUE KEY \`uk\_batch\_code\` (\`batch\_code\`),

  KEY \`idx\_product\_id\` (\`product\_id\`),

  KEY \`idx\_inbound\_order\_id\` (\`inbound\_order\_id\`),

  CONSTRAINT \`fk\_inventory\_product\` FOREIGN KEY (\`product\_id\`) REFERENCES \`products\` (\`id\`) ON DELETE RESTRICT,

  CONSTRAINT \`fk\_inventory\_inbound\_order\` FOREIGN KEY (\`inbound\_order\_id\`) REFERENCES \`inbound\_orders\` (\`id\`) ON DELETE SET NULL

) COMMENT='库存及批次表';

 

\-- \=============================================

\-- 7\. 异动日志表

\-- 记录每一次库存变动（入库、出库、调整），用于审计和报表。

\[cite\_start\]-- App的扫码操作会填充此表，为数据统计提供依据 \[cite: 25\]。

\-- \=============================================

CREATE TABLE \`transaction\_logs\` (

  \`id\` BIGINT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

  \`inventory\_id\` INT UNSIGNED NOT NULL COMMENT '关联的库存批次ID',

  \[cite\_start\]\`user\_id\` INT UNSIGNED NOT NULL COMMENT '执行此操作的用户 (操作员) \[cite: 18\]',

  \`type\` ENUM('入库', '出库', '调整') NOT NULL COMMENT '异动类型',

  \`quantity\_change\` INT NOT NULL COMMENT '库存变化数量 (+表示入库, \-表示出库)',

  \`quantity\_after\_transaction\` INT NOT NULL COMMENT '本次异动后, 批次的库存数量',

  \`notes\` VARCHAR(255) COMMENT '可选备注, 例如: 盘点调整原因',

  \`created\_at\` TIMESTAMP NOT NULL DEFAULT CURRENT\_TIMESTAMP,

  KEY \`idx\_inventory\_id\` (\`inventory\_id\`),

  KEY \`idx\_user\_id\` (\`user\_id\`),

  CONSTRAINT \`fk\_logs\_inventory\` FOREIGN KEY (\`inventory\_id\`) REFERENCES \`inventory\` (\`id\`) ON DELETE RESTRICT,

  CONSTRAINT \`fk\_logs\_user\` FOREIGN KEY (\`user\_id\`) REFERENCES \`users\` (\`id\`) ON DELETE RESTRICT

) COMMENT='库存异动日志表';
