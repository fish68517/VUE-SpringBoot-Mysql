# 前端响应式设计指南

## 概述

本文档描述了广州旅游管理系统前端的响应式设计实现，确保系统在各种屏幕尺寸上都能提供良好的用户体验。

## 响应式断点

系统采用以下响应式断点来适配不同的设备：

| 断点 | 屏幕宽度 | 设备类型 | 说明 |
|------|---------|---------|------|
| 超大屏 | > 1024px | 桌面电脑 | 完整布局，所有功能可见 |
| 大屏 | 768px - 1024px | 平板电脑 | 优化的布局，适当缩小间距 |
| 中屏 | 480px - 768px | 大型手机 | 单列布局，优化触摸交互 |
| 小屏 | 360px - 480px | 小型手机 | 最小化布局，简化导航 |
| 超小屏 | < 360px | 极小设备 | 极简布局 |

## 通用响应式设计原则

### 1. 字体大小缩放

```css
/* 标题 */
h1: 28px → 24px → 20px → 18px → 16px
h2: 24px → 20px → 18px → 16px → 14px
h3: 20px → 18px → 16px → 14px → 12px

/* 正文 */
p: 14px → 13px → 12px → 11px → 10px
```

### 2. 间距调整

```css
/* 容器内边距 */
20px → 15px → 12px → 10px → 8px

/* 元素间距 */
20px → 15px → 10px → 8px → 6px
```

### 3. 布局变化

- **超大屏**: 多列布局，充分利用空间
- **大屏**: 优化的多列布局
- **中屏**: 2列或单列布局
- **小屏**: 单列布局，优化触摸
- **超小屏**: 极简单列布局

### 4. 图片处理

- 使用 `object-fit: cover` 保持宽高比
- 响应式高度调整
- 移动端优化加载

### 5. 触摸交互

- 按钮最小高度: 44px（移动端标准）
- 点击区域最小: 44x44px
- 间距优化便于触摸

## 性能优化建议

### 1. 图片优化
- 使用 WebP 格式（带回退）
- 实现图片懒加载
- 根据屏幕尺寸加载不同分辨率

### 2. CSS 优化
- 使用 CSS 变量管理响应式值
- 最小化媒体查询重复
- 使用 CSS Grid 和 Flexbox

### 3. JavaScript 优化
- 避免在移动端加载不必要的脚本
- 使用事件委托减少监听器
- 优化触摸事件处理

## 常见问题

### Q: 如何测试响应式设计？
A: 使用浏览器开发者工具的响应式设计模式，或在实际设备上测试。

### Q: 如何添加新的响应式断点？
A: 在相应的 Vue 组件中添加新的 `@media` 查询，遵循现有的命名约定。

### Q: 如何处理横屏和竖屏？
A: 使用 `orientation` 媒体查询，或通过 JavaScript 检测方向变化。

## 最佳实践

1. **移动优先**: 从小屏幕开始设计，逐步增加功能
2. **灵活布局**: 使用 Flexbox 和 Grid 实现灵活布局
3. **相对单位**: 优先使用 em、rem 而不是 px
4. **触摸友好**: 确保按钮和链接足够大，便于触摸
5. **性能优先**: 优化加载时间，特别是在移动网络上
6. **测试充分**: 在多种设备和浏览器上测试
