在线投稿系统需满足作者、编辑、审稿人、系统管理员四类角色的使用需求，所有角色均支持注册、登录、退出登录功能，各角色核心功能如下：

**系统管理员：**

①个人信息管理：修改账号密码、联系方式等基本信息；

②用户管理：审核编辑、审稿人注册信息（批准 / 驳回），查询、修改、删除各类用户账号，分配用户权限；

③栏目管理：对投稿栏目（如学术论文、科普文章、文学作品等）进行增删改查操作，设置栏目投稿要求；

④系统配置：设置稿件审核流程、投稿格式要求、通知模板等系统参数，管理系统日志；

⑤数据统计：统计各栏目投稿量、审核通过率、用户活跃度等数据，生成统计报表。

**编辑：**

①个人信息管理：修改个人基本信息、工作邮箱等；

②稿件管理：接收作者投稿，进行初审（通过 / 驳回 / 退修），将合格稿件分配给对应领域的审稿人，跟踪审稿进度，查看审稿意见；

③稿件编辑：对通过终审的稿件进行格式排版、内容校对，生成录用通知；

④沟通管理：回复作者咨询，发送稿件审核结果通知，与审稿人进行工作沟通。

**审稿人：**

①个人信息管理：完善专业领域、研究方向等信息，修改登录密码；

②审稿任务管理：接收编辑分配的审稿任务（接受 / 拒绝），在线查看稿件内容，提交审稿意见（同意录用 / 修改后录用 / 拒绝录用）及评分；

③历史记录查询：查询过往审稿任务及对应的审稿意见。

**作者：**

①个人信息管理：维护个人基本信息、学术成果、联系方式等，修改账号密码；

②稿件管理：按栏目提交稿件（上传文档、填写稿件信息），查看稿件审核进度及各阶段意见，接收退修通知并上传修改后的稿件，查询历史投稿记录；

③投稿操作：对未审核的稿件进行修改、撤回操作，查看录用/驳回通知；

④消息中心：接收系统通知、编辑沟通消息等，查看消息历史。

技术栈和技术要求

请为我生成一个基于 Spring Boot 3.x + Vue 3 + MySQL 的前后端分离 Web 系统。

要求如下：

1. 后端不要使用 JWT、Spring Security、Sa-Token、RBAC 等权限框架。
2. 登录密码不加密，直接明文校验。
3. 图片上传后直接保存到 Spring Boot 项目根目录 image 文件夹下。
4. 数据库中只保存图片文件名或相对路径，不保存图片二进制。表的设计需要修改数据库外键为“级联删除”ON DELETE CASCADE
5. 需要配置静态资源映射，使 /image/** 可以直接访问图片。
6. 需要配置全局 CORS，允许所有前端来源访问所有接口和图片资源。
7. 前端使用 Vue 3 + Vue Router + Pinia + Axios + Element Plus。
8. 作者端采用顶部导航栏 + 内容区布局。
9. 管理员、编辑、审稿人采用左侧边栏 + 内容区布局。
10. 登录后根据角色跳转到不同布局页面。
11. 页面风格简洁，适合本科毕业设计展示。
12. 代码结构保持清晰，按 controller、service、mapper、entity、dto、vo、config 分层。后端可以从HttpSession中或者当前登录用户的UserId
13. 优先保证系统可运行、页面可访问、接口可联调，不要过度设计。
14. 页面所有的文本采用：中文简体，代码注释和调试日志也采用 中文简体
